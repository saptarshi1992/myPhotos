About

Hi, I'm Saptarshi — a web developer from India with a strong background in backend development. Recently, I’ve been diving deeper into the frontend world, and CSS has become a growing area of interest for me.

While organizing my travel photo collection, I was inspired to build something creative using CSS. I came across the CSS Carousel Gallery by Adam Argyle , and thought it would be a great opportunity to try out this new CSS feature.

I decided to build a carousel-within-a-carousel structure — a horizontal carousel for different countries I've visited, and a vertical carousel within each for the photos. It’s my way of exploring CSS while organizing my memories from travel.

Along the way, I also experimented with position-anchor and position-area, which let me position elements independently of their parent containers. It was really exciting to explore how much control CSS now offers.

Challenges I Faced

One tricky part was managing opacity. I wanted to reduce the opacity of the next photo in the stack without showing the title and description — but CSS selectors got messy fast, and I kept accidentally breaking the layout.

Another issue was browser compatibility. Some features didn’t behave as expected in unsupported browsers. While scrolling still worked, some content got misaligned. I plan to improve fallback support later, but for now, I’ve added a warning banner for unsupported browsers.

Overall, while CSS Carousels are fun to experiment with, I wouldn’t necessarily recommend them for serious photo galleries — they can get a bit clunky. Still, this was a great learning experience, and I’m happy I pushed myself to try something new.
